kubectl 命令完整手册

一、基础查看命令

1. get - 列出资源

最常用的命令,列出资源。

2. describe - 查看资源详情

何时用:

  • Pod 启动失败,查原因
  • 看资源的完整配置和运行状态
  • 查 Events(最下面)定位问题

3. logs - 查看日志

4. top - 查看资源使用

需要先装 metrics-server。

二、创建和删除资源

5. apply - 声明式创建/更新(推荐)

6. create - 命令式创建

7. run - 快速跑 Pod

8. delete - 删除资源

三、更新和修改资源

9. set - 修改资源的特定字段(快捷)

10. edit - 在线编辑(打开编辑器)

11. patch - 精确修改(最灵活)

12. replace - 替换整个资源

区别:

  • apply:声明式,保留未声明字段
  • replace:完全替换,未声明字段会丢失

13. scale - 扩缩容

14. rollout - 滚动管理

四、标签和污点

15. label - 管理标签

16. annotate - 管理注解

用法和 label 类似,注解是给工具/系统用的。

17. taint - 管理污点

五、调试和交互

18. exec - 进入容器执行命令

19. cp - 在容器和本地间复制文件

20. port-forward - 端口转发

把集群内服务临时映射到本地

21. proxy - 代理到 API server

22. debug - 调试工具(1.25+)

六、集群和节点管理

23. cluster-info - 集群信息

24. cordon / uncordon - 节点暂停调度

25. drain - 驱逐 Pod(节点维护)

七、配置和上下文

26. config - kubeconfig 管理

八、高级技巧

27. 格式化输出

28. 标签选择器

29. wait - 等待条件

30. explain - 查资源字段文档

不懂 YAML 某字段怎么写?看文档

十、修改资源的四种方式对比

方式 优点 缺点 适用
set 简单直接 只能改预定义字段 快速修改
edit 交互式,能看到全文 改错容易出问题 临时调整
patch 精确,能脚本化 语法稍复杂 自动化
apply 声明式,可追溯 要维护 YAML 文件 GitOps
helm 最规范 要改 values 文件 Helm 安装的组件
